Transform Spaces¶

Objects can be manipulated in various co-ordinate spaces.

class pygly.object_space.ObjectSpace(transform)[source]¶

Provides transform methods for manipulating objects within object space co-ordinates.

Object space is defined as being relative to the object itself. The translation and orientation of the X,Y,Z axis remain fixed to the object as it moves.

__init__(transform)[source]¶

Constructs an ObjectSpace object that interacts with the specified transform object.

Args:
transform: The transform to control.
rotate_quaternion(quat)[source]¶

Rotates the transform by the specified orientation.

rotate_x(radians)[source]¶

Pitch the transform about it’s X axis.

Note

Amount > 0 == pitch down Amount < 0 == pitch up

rotate_y(radians)[source]¶

Yaw the transform about it’s Y axis.

Note

Amount > 0 == yaw right. Amount < 0 == yaw left.

rotate_z(radians)[source]¶

Roll the transform about it’s Z axis.

Note

Amount > 0 == roll left. Amount < 0 == roll right.

translate(vector)[source]¶

Translates the transform locally.

The vector will have the node’s current orientation applied to it and then be added to the translation.

translation[source]¶
x[source]¶

Returns the object’s local X axis.

This is the X axis rotated by the objects orientation.

Note

This is NOT the world orientation. To get inertial X axis, simply use [1.0, 0.0, 0.0].

y[source]¶

Returns the object’s local Y axis.

This is the Y axis rotated by the objects orientation.

Note

This is NOT the world orientation. To get inertial Y axis, simply use [0.0, 1.0, 0.0].

z[source]¶

Returns the object’s local Z axis.

This is the Z axis rotated by the objects orientation.

Note

This is NOT the world orientation. To get inertial Z axis, simply use [0.0, 0.0, 1.0].

class pygly.inertial_space.InertialSpace(transform)[source]¶

Provides transform methods for manipulating objects within inertial space co-ordinates.

Inertial space is defined as using the traslation of the object but the rotation remains relative to the parent.
